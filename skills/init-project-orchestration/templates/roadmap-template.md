## Notes

- **Phases are sequential** — complete one before starting the next.
- **Checkbox subtasks** make progress trackable from `state.md`.
- **Deliverables are concrete** — "Draft of Section 3" not "Work on paper".
- **Phase 0 is always pre-checked** — the skill creates it as complete.
- **Add phases as needed** — this is a living document, not a fixed plan.
