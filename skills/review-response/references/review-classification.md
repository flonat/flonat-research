# Review Comment Classification

Systematic classification criteria for reviewer comments. Use this to quickly categorise and prioritise each comment before developing response strategies.

## Classification System

### 1. Major Issues

**Definition:** Core problems with methodology, experimental design, results interpretation, or contribution claims. Requires substantive revision, additional analysis, or new experiments.

**Characteristics:**
- Questions the validity of the research method
- Requests additional key experiments or comparisons
- Points out logical problems in results interpretation
- Questions the novelty or contribution

**Keyword signals:**
- "major concern"
- "fundamental issue"
- "missing experiments"
- "insufficient evidence"
- "not convincing"

**Example:**
> "The authors should compare their method with recent work [X]. Without this comparison, it's difficult to assess the true contribution."

**Recommended strategy:** Accept + Experiment (accept the point and provide additional evidence)

---

### 2. Minor Issues

**Definition:** Points requiring clarification, additional explanation, or modest revision. Does not affect the core contribution.

**Characteristics:**
- Requests clarification of specific details
- Suggests improved presentation or organisation
- Requests additional related work discussion
- Points out figure/table improvements

**Keyword signals:**
- "minor concern"
- "could be improved"
- "please clarify"
- "it would be better"
- "suggestion"

**Example:**
> "The notation in Section 3.2 is confusing. Please clarify the meaning of variable alpha."

**Recommended strategy:** Accept + Clarify (accept and provide clarification)

---

### 3. Typos / Formatting

**Definition:** Surface-level errors — spelling, grammar, formatting inconsistencies, reference errors.

**Characteristics:**
- Spelling errors
- Grammar issues
- Inconsistent citation formatting
- Figure/table labelling errors

**Keyword signals:**
- "typo"
- "grammar"
- "formatting"
- "inconsistent"
- "Figure X should be"

**Example:**
> "Page 5, line 3: 'recieve' should be 'receive'."

**Recommended strategy:** Accept (fix directly and thank the reviewer)

---

### 4. Misunderstandings

**Definition:** The reviewer misread or missed content that is already in the paper. Requires polite clarification.

**Characteristics:**
- Reviewer claims something is missing that is already present
- Reviewer misunderstands the method or results
- Reviewer has an incorrect understanding of a concept

**Keyword signals:**
- "The authors did not..." (but they did)
- "It is unclear..." (but it is stated)
- "Why don't the authors..." (but they already did)

**Example:**
> "The authors did not compare with baseline X."
> (But Table 2 already includes this comparison)

**Recommended strategy:** Clarify (politely point to existing content)

---

## Classification Workflow

1. **First pass** — Skim all comments, flag keywords
2. **Classify** — Label each comment: Major / Minor / Typo / Misunderstanding
3. **Prioritise** — Order by: Major > Misunderstanding > Minor > Typo
4. **Assign strategy** — Match each comment to a response strategy

## Priority Guidance

**High priority (address first):**
- Major Issues — require substantive revision or new analysis
- Misunderstandings — need clarification to prevent the misunderstanding deepening

**Medium priority:**
- Minor Issues — improvements that don't affect the core contribution

**Low priority:**
- Typos / Formatting — fix directly, confirm in rebuttal

## Important Notes

- **Never ignore any comment** — even small typos should be acknowledged in the rebuttal
- **Be polite about misunderstandings** — use "We apologise for the confusion" not "The reviewer misunderstood"
- **Major Issues need substantive responses** — don't just clarify; show action (new experiments, revised analysis, etc.)
