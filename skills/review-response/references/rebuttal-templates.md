# Rebuttal Templates

Ready-to-use templates for structuring rebuttal documents.

## Standard Rebuttal Structure

```markdown
# Response to Reviewers

We sincerely thank all reviewers for their valuable feedback and constructive
suggestions. We have carefully addressed all comments and made substantial
revisions to improve the manuscript. Below, we provide detailed responses to
each reviewer's comments.

---

## Response to Reviewer 1

We thank Reviewer 1 for the thorough review and insightful comments.

### Major Comments

**Comment 1.1**: [Reviewer's original comment, verbatim]

**Response**: [Substantive reply]

**Changes**: [Specific modifications with section/page/table references]

---

**Comment 1.2**: [Reviewer's original comment]

**Response**: [Reply]

**Changes**: [Locations]

---

### Minor Comments

**Comment 1.3**: [Comment]

**Response**: [Reply]

**Changes**: [Locations]

---

## Response to Reviewer 2

[Same structure]

---

## Summary of Major Changes

In response to the reviewers' feedback, we have made the following major revisions:

1. [Change 1] (addressing Reviewer X, Comment Y)
2. [Change 2] (addressing Reviewers X and Z)
3. [Change 3] (addressing all reviewers)

We believe these revisions have significantly strengthened the manuscript and
addressed all concerns raised by the reviewers.
```

## Opening Paragraphs

### Standard thanks
```
We sincerely thank all reviewers for their valuable feedback and constructive
suggestions. We have carefully addressed all comments and made substantial
revisions to improve the manuscript.
```

### Emphasis on improvements
```
We are grateful to the reviewers for their thorough and insightful comments.
Their feedback has helped us significantly improve the quality and clarity of
our work. We have made extensive revisions to address all concerns.
```

### Highlighting key changes
```
We thank the reviewers for their careful evaluation. We have addressed all
comments and made major revisions, including [key improvement 1],
[key improvement 2], and [key improvement 3].
```

## Per-Strategy Response Templates

### Accept — simple
```
We thank the reviewer for this valuable suggestion. We have [specific change].
[Location].
```

### Accept — with expansion
```
We appreciate this insightful comment. We agree that [issue] is important.
We have [action 1] and [action 2]. These changes are reflected in [location 1]
and [location 2].
```

### Defend — polite
```
We appreciate the reviewer's concern. However, we respectfully note that
[our approach] is motivated by [reason]. Specifically, [explanation].
We have added this clarification to Section [X].
```

### Clarify — point to existing content
```
We thank the reviewer for raising this point. We would like to respectfully
clarify that [existing content]. This is discussed in [location]. To make this
clearer, we have [improvement].
```

### Experiment — completed
```
We thank the reviewer for this excellent suggestion. We have conducted
additional [analysis] on [content]. The results show that [findings].
These new results have been added to [location].
```

### Experiment — in progress
```
We appreciate this valuable suggestion. We agree that [importance]. We are
currently conducting [work] and will include the results in the revised
manuscript.
```

## Special Situations

### Multiple reviewers raise the same issue
```
We thank Reviewers [X] and [Y] for raising this important point. We agree
that [issue]. We have [action]. These changes address both reviewers' concerns
and are reflected in [location].
```

### Reviewers contradict each other
```
We appreciate both reviewers' perspectives on [topic]. Reviewer [X] suggests
[suggestion X], while Reviewer [Y] suggests [suggestion Y]. After careful
consideration, we have [chosen approach] because [reasons]. We believe this
addresses both reviewers' core concerns.
```

### Cannot fully satisfy a request
```
We thank the reviewer for this suggestion. While we agree that [value of
suggestion], [constraint preventing full implementation]. However, we have
[partial implementation or alternative], which [benefit]. We acknowledge this
limitation in [location] and discuss it as future work.
```

## Closing Templates

### Per-reviewer closing
```
We hope our responses and revisions have adequately addressed all of your
concerns. We are grateful for your valuable feedback.
```

### Overall closing — summary of changes
```
## Summary of Major Changes

In response to the reviewers' feedback, we have made the following major revisions:

1. [Change 1] (Reviewers [X, Y])
2. [Change 2] (Reviewer [Z])
3. [Change 3] (All reviewers)

We believe these revisions have significantly strengthened the manuscript.
```

## Tone Checklist (run before submitting)

### Must include:
- [ ] Every response opens with thanks
- [ ] Uses "We" not "I"
- [ ] Specific location references (section, page, table, figure)
- [ ] Concrete description of changes made
- [ ] Professional, respectful tone throughout

### Must avoid:
- [ ] No "The reviewer is wrong" or similar
- [ ] No "Obviously", "Clearly", or condescending language
- [ ] No defensive or aggressive tone
- [ ] No vague promises ("We will..." without specifics)
- [ ] No ignored comments — every point gets a response
